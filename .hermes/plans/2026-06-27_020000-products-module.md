# Module Products + Categories — Plan

> **Goal:** CRUD produits et catégories avec gestion de stock. Admin only pour création/modification, public pour consultation.

**Architecture:** 2 modules NestJS (ProductsModule, CategoriesModule). Products lié à Category via categoryId. Stock géré sur Product.stockQuantity. Admin protégé par JwtAuthGuard + @Roles(ADMIN).

**Files:**
- `src/products/products.module.ts`
- `src/products/products.service.ts` + `.spec.ts`
- `src/products/products.controller.ts` + `.spec.ts`
- `src/products/dto/create-product.dto.ts`
- `src/products/dto/update-product.dto.ts`
- `src/categories/categories.module.ts`
- `src/categories/categories.service.ts` + `.spec.ts`
- `src/categories/categories.controller.ts` + `.spec.ts`
- `src/categories/dto/create-category.dto.ts`

**Tasks:**
1. CategoriesModule: service + controller (CRUD, public GET, admin POST/PUT/DELETE)
2. ProductsModule: service + controller (CRUD, public GET, admin POST/PUT/DELETE, stock)
3. Tests (10+) + tsc + commit
